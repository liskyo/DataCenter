# 閒置實體伺服器：Agent 測試與部署實戰教學
**日期：** 2026-03-23

太棒了！能夠親自用一台閒置電腦或伺服器來做實驗，絕對是打通任督二脈最快的方法。這份教學將帶你一步步將那台閒置伺服器變成「真實監控節點」，並將它的體溫與心跳即時投射到你的 DataCenter 儀表板上。

---

## 前置準備：確認兩台機器的角色

在開始之前，請先確認兩台機器的定位：
1. **主控電腦 (Dashboard 座艙)**：
   - 特徵：目前你正用來打開這份文件、跑緊 `start.bat`、打開著 `localhost:3001` 的那台電腦。
   - 任務：我們要先查出這台主機在你們辦公室區域網路內的**真實 IP (例如 `192.168.0.100`)**。
     > *小訣竅：打開 Windows 終端機，輸入 `ipconfig`，找到 IPv4 位址並記下來。*
     
2. **閒置伺服器 (被監控端 Agent)**：
   - 特徵：你手邊多出來的那台實體電腦，作業系統是 Linux 或 Windows 皆可。
   - 任務：跑小幫手程式，負責向主機丟資料。

---

## 第一階段：在閒置伺服器上裝配套件

請切換到你那台**閒置伺服器**的終端機或遠端桌面，依照以下步驟操作：

### 1. 安裝環境與依賴套件
確認該伺服器上已經有安裝 Python (推薦版本 3.8 以上)。
接著，打開終端機，安裝我們 Agent 需要使用的兩個開源輕量級套件：
```bash
# 對應抓取硬體感測器 (psutil) 與發送網路請求 (requests)
pip install psutil requests
```

### 2. 佈署上傳 Agent 程式
將我們剛剛在主控電腦上寫好的 **`client_agent.py`** 這個檔案，透過 USB、雲端硬碟或 SCP，**複製並貼進那台閒置伺服器中**的任何一個資料夾（例如放桌面上）。

### 3. 設定 Agent 瞄準中心目標
用記事本或任何編輯器打開那台閒置伺服器上的 `client_agent.py`，更改最上面的大寫變數：

```python
# 1. 這台真實機器的名稱 (請隨意取，例如 SERVER-001)
MY_SERVER_ID = "SERVER-001"

# 2. 控制中心的真實 IP 位址 (請填入主控電腦的 ipconfig IPv4 位址)
# 假設主控電腦是 192.168.0.100，就要這樣填：
DATACENTER_API_URL = "http://192.168.0.100:8000/ingest"

# 3. 回報頻率 (秒數不變，維持 5 秒)
REPORT_INTERVAL_SECONDS = 5
```
*(存檔關閉)*

---

## 第二階段：進行發射測試！

### 步驟一：讓 Dashboard 切換模式
請在**主控電腦**上打開瀏覽器 (`http://localhost:3001`)：
1. 看向畫面的右上角。
2. 點擊 **`SIMULATION MODE`** 按鈕。
3. 它會切換成 **`LIVE DATA MODE`**。
   *(此時，因為後端模擬器被我們關掉了，所以所有的圖表跟 3D 畫面都會在幾秒後變成完全死寂的 `OFFLINE / NO SIGNAL`。這代表系統準備好迎接真實資料了！)*

### 步驟二：發射真實訊號
回到你的**閒置伺服器**終端機，啟動我們的小幫手：
```bash
# 在放置 client_agent.py 的資料夾目錄下執行：
python client_agent.py
```
> 如果一切順利，你應該會在終端機看到：
> `✅ 成功回報 - CPU: 12.5% | Temp: 38.6°C`
> 並且每 5 秒噴出一行新的綠色成功字眼！

### 步驟三：見證奇蹟的時刻
立刻抬頭看向你的**主控電腦螢幕**！
原本全部灰暗的儀表板上，那個叫做 `"SERVER-001"` 的小區塊（或 3D 數位機櫃）會瞬間亮起。
你會看到：
1. 它的 CPU 數字完全依照你閒置伺服器目前的真實負載在跳動。
2. *進階測試：* 嘗試在該台閒置伺服器上隨便開幾個大型軟體（或是跑個 CPU 壓力測試燒機），當它的物理溫度或負載超過 `60` 次或 `85` 臨界點時，主控電腦上的模型就會如預期般立刻轉成紅色火焰告警狀態！

恭喜你！這就是跨機器、免改中心系統、極具擴展性的**現代機房真實監控流程**！
