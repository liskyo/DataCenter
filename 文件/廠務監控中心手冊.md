# 廠務監控中心 (Facility Monitoring Center) 手冊

## 1. 系統概述 (Overview)
廠務監控中心（Facility Panel）是 3D 數據中心 (DataCenter Digital Twin) 的核心子系統之一。其主要功能是從宏觀的廠務基礎設施 (Gray Space) 視角，提供實時的能源效率、環境參數、電力配置、冷卻系統狀態以及實體安全的綜合數據看板。

本手冊旨在說明該中心目前監控的各大指標、背後的商業與維運邏輯，以及未來若要導入真實環境時，應如何與實體設備進行對接 (Integration)。

---

## 2. 監控項目與對接邏輯 (Monitoring Items & Integration Logic)

目前的監控面板分為五大核心區塊，每一區塊皆對應資料中心營運的關鍵指標：

### 2.1 永續與能源效率 (Sustainability & KPIs)
本區塊主要監控資料中心的綠能指標與能源使用效率，為 ESG 報告的核心數據。

*   **PUE (電力使用效率 / Power Usage Effectiveness)**
    *   **定義**：(資料中心總耗電量) / (IT 設備總耗電量)。越接近 1.0 代表冷卻與廠務耗損越少，效率越好。
    *   **對接設備**：總盤數位電表 (Main Power Meter)、UPS 輸出端電表、列頭櫃 (Floor PDU) 總表。
    *   **整合邏輯**：系統需定期 (例如每分鐘) 從 BMS 系統或電表抓取即時 kW/kWh 數據，進行即時相除與平滑化運算。
*   **WUE (用水使用效率 / Water Usage Effectiveness)**
    *   **定義**：(資料中心總耗水量) / (IT 設備總耗電量)，單位為 L/kWh。
    *   **對接設備**：冷卻水塔與機房主水管上的數位水表 (Digital Water Flow Meters)。
*   **ERF (能源再利用率 / Energy Reuse Factor)**
    *   **定義**：(重新利用的能源) / (資料中心總耗能)。例如將伺服器廢熱導出用於辦公室供暖。
    *   **對接設備**：熱回收系統 (Heat Recovery System) 之熱能計 (Heat Meters)。

### 2.2 環境監控 (Environmental)
監控機房內的微氣候，防止過熱、靜電或漏水導致硬體損壞。

*   **RACK-INLET TEMP (機櫃進風溫度)**
    *   **對接設備**：部署於冷通道或機櫃前門的 IoT 溫度感測器 (Temperature Sensors)。
    *   **整合邏輯**：透過 MQTT 或 SNMP 採集各機櫃進風口的溫度，面板上呈現的是全區平均值或最熱點 (Hot Spot) 溫度。若高於 27°C 通常會觸發第一級警告。
*   **HUMIDITY (相對濕度)**
    *   **對接設備**：IoT 濕度感測器。
    *   **整合邏輯**：過低（<30%）易生靜電，過高（>60%）易致金屬微短路。系統應設置上下限閾值警報。
*   **AIR DIFF PRESSURE (空調微差壓 / ΔP)**
    *   **對接設備**：微差壓計 (Differential Pressure Sensors)，通常一端置於冷通道高架地板下，一端置於熱通道。
    *   **整合邏輯**：維持冷通道的微正壓（例如 0.05 iwg），確保冷風能被強迫灌入伺服器內，避免熱風回流。
*   **LEAK DETECTION (漏液偵測)**
    *   **對接設備**：漏水偵測繩 (Leak Detection Cables) 或漏液點感測器，鋪設於冰水管路下方、高架地板下及 CDU 設備旁。
    *   **整合邏輯**：感測器偵測到導電液體時電阻改變，Controller 會送出 Dry Contact 訊號或 Modbus 警報。系統收到後立即將狀態改為紅色的 `DETECTED` 並觸發聲光警報。

### 2.3 電力鏈 (Power Infrastructure)
追蹤從市電端到機櫃端的電力供應健康度。

*   **TOTAL UPS LOAD (不斷電系統總負載)**
    *   **對接設備**：不斷電系統 (UPS)。
    *   **整合邏輯**：讀取 UPS 的即時輸出功率 (kW 或 kVA) 與其額定容量的百分比。當負載超過 80% 時應發出擴容警告。
*   **Floor PDU-A / PDU-B (列頭櫃負載)**
    *   **對接設備**：列頭櫃 (Floor Power Distribution Units) 內的智慧電表。
    *   **整合邏輯**：A/B 雙回路供電是資料中心標配。系統需確保 A/B 兩路負載均衡，且即使 A 路斷電，B 路也能獨自承載 100% 負載不跳機。
*   **UPS Battery (電池電量與狀態)**
    *   **對接設備**：電池監控系統 (BMS / Battery Monitoring System)。
    *   **整合邏輯**：平時狀態為 `Float` (浮充)，若市電異常則轉為 `Discharging` (放電中)，系統需依據負載計算剩餘放電時間 (Run Time)。

### 2.4 冷卻系統 (Thermal Management)
負責帶走 IT 設備產生的大量熱能，為機房最耗電的附屬設施。

*   **CHILLER PUMP SPEED (冰水泵轉速)**
    *   **對接設備**：冰水泵上的變頻器 (VFD / Variable Frequency Drive)。
    *   **整合邏輯**：依據機房熱負荷，控制器會動態調整水泵轉速 (Hz)。面板透過 Modbus 讀取即時運轉頻率。
*   **SECONDARY FLOW RATE (二次側流量)**
    *   **對接設備**：超音波流量計或電磁流量計 (Flow Meters)。
    *   **整合邏輯**：監控進入機房冷卻單元 (如 CRAH 或 CDU) 的即時水流量 (GPM - Gallons Per Minute)。
*   **Supply Return Delta-T (供回水溫差)**
    *   **對接設備**：主冰水管路上的溫度計。
    *   **整合邏輯**：取得「供水溫度 (Supply)」(通常約 10~15°C) 與「回水溫度 (Return)」。兩者的溫差 (ΔT) 越大，代表熱交換效率越好；若 ΔT 過小，可能代表水流量過大或設備未滿載 (Bypass 問題)。

### 2.5 實體安全 (Physical Context)
監控機房的門禁與人員動態。

*   **CCTV (監視器)**
    *   **對接設備**：網路攝影機 (IP Cameras) 或 NVR 系統。
    *   **整合邏輯**：透過 RTSP / WebRTC 協定，將監視器的即時串流畫面直接嵌入至前端面板。
*   **Main Gate / Rack Door (主大門與機櫃門)**
    *   **對接設備**：門禁控制器 (Access Control System)、機櫃電子鎖 (Smart Rack Locks)。
    *   **整合邏輯**：接收門禁系統的 Event Stream (Webhook 或 Message Queue)。平時顯示 `SECURED`，若被強制開啟或未關妥則閃爍提示 `OPEN`。
*   **Motion Sensor (動靜感測器)**
    *   **對接設備**：紅外線防盜感測器 (PIR Sensors)。
    *   **整合邏輯**：非授權時間內若感測到人員移動，立刻連動 CCTV 彈出畫面並發布警報。

### 2.6 消防與工安安全 (Fire & Life Safety)
保障機房免受火災威脅的最核心防線。

*   **VESDA 極早期煙霧預警系統**
    *   **對接設備**：空氣抽取式煙霧探測器。
    *   **整合邏輯**：在產生明火前，只要空氣中有微量塑膠燒焦微粒即可觸發預警，通常比傳統煙霧探測器快上數小時。
*   **FM200 GAS (氣體滅火壓力)**
    *   **對接設備**：氣體滅火鋼瓶壓力計。
    *   **整合邏輯**：監控鋼瓶壓力是否在正常值（例如 360 psi），避免氣體洩漏導致真正發生火災時無法滅火。
*   **EPO (緊急斷電開關)**
    *   **對接設備**：機房門口的緊急斷電按鈕。
    *   **整合邏輯**：一旦觸發將瞬間切斷機房所有電力。平時應嚴密監控是否處於 `ARMED` (武裝未觸發) 狀態。

### 2.7 備援發電系統 (Backup Power)
市電中斷時，維持機房運作的最後防線。

*   **ATS (自動切換開關)**
    *   **對接設備**：ATS/STS 轉換開關。
    *   **整合邏輯**：監控目前機房是由市電 (Utility) 還是柴油發電機 (Generator) 供電。
*   **DIESEL TANK (柴油發電機油槽)**
    *   **對接設備**：地下油槽或日用油箱的液位計。
    *   **整合邏輯**：監控剩餘油量百分比，並依據當前負載計算出還能支撐多少小時（Run Time），做為請求油罐車補給的依據。

### 2.8 資源容量管理 (Capacity Planning)
輔助 IT 與維運主管進行擴展與資源調度的決策指標。

*   **RACK SPACE (機櫃 U 數使用率)**
    *   **對接設備**：DCIM 資產管理資料庫。
    *   **整合邏輯**：統計全機房有多少機櫃空間已被佔用，可用於評估是否需要擴建或整理閒置設備。
*   **POWER CAPACITY (可用電力裕度)**
    *   **對接設備**：總電力容量設定檔與即時總耗電量。
    *   **整合邏輯**：顯示目前設計總容量還剩餘多少 kW，以評估是否能再容納新的高密度伺服器（如 AI GPU 伺服器）。

## 3. 當前系統實作狀態 (Current Implementation Status)

在目前的專案中（`frontend/src/app/facility/page.tsx`），上述所有的數據皆處於 **模擬階段 (Mocked State)**。

*   **模擬機制**：透過 React 的 `useEffect` 與 `setInterval`，每 2 秒鐘自動對靜態數值加上一個隨機的亂數 (Random noise)，營造出數據即時跳動的視覺效果。
*   **未來升級指南**：
    若要接入真實數據，請將前端的計時器移除，並替換為：
    1.  建立 **WebSocket / Server-Sent Events (SSE)** 連線，訂閱後端 IoT 平台 (如 ThingsBoard) 的串流服務。
    2.  將收到的 JSON Payload (例如 `{ "pue": 1.25, "temp": 24.1, "leak": false }`) 解析後，直接寫入 React 的 `useState` 或交由 Zustand 統一管理狀態，UI 即可瞬間完成數據熱更新與真實設備連動。
