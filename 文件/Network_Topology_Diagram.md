# 機房實體網路拓撲圖 (Network Topology)
**日期：** 2026-03-23

你完全說對了！最簡單、也最常見的物理連接方式，就是準備一台 **網路交換器 (Network Switch)**，然後買幾條網路線把大家串在一起。

以下是這個監控系統最標準的實體層 (Physical Layer) 架構圖：

```text
                                  【外部網際網路 / Router】(可選: 用來自動發 IP 或上網)
                                            │
                                            │ (網路線)
                                            ▼
┌────────────────────────────────────────────────────────────────────────┐
│                        網路交換器 (Network Switch)                       │
│      [ Port 1 ]      [ Port 2 ]      [ Port 3 ]  ...   [ Port 24 ]     │
└────────────────────────────────────────────────────────────────────────┘
            │               │               │                  │
   (網路線) │               │               │                  │
            ▼               ▼               ▼                  ▼
      ┌──────────┐    ┌──────────┐    ┌──────────┐       ┌──────────┐
      │ 中控主機 │    │ 實體伺服器│    │ 實體伺服器│       │ 新買的機器│
      │Dashboard │    │ SERVER-1 │    │ SERVER-2 │       │ SERVER-3 │
      └──────────┘    └──────────┘    └──────────┘       └──────────┘
      192.168.1.10    192.168.1.11    192.168.1.12       192.168.1.13
     (負責接收訊號)  (跑 client_agent 發送訊號 ➔ 指向 192.168.1.10:29092 或 9000)
```

## 設定重點
1. **硬體接法**：沒錯，就是買一台 Switch，把「你看 Dashboard 的這台電腦」跟「機房裡的所有伺服器」全部用網路線插上去。
2. **網路設定 (IP)**：
   - 只要確保他們全都在同一個「區域網路 (LAN)」底下。例如大家的 IP 前三段都是 `192.168.1.X`。
   - **中控主機 (Dashboard)** 的 IP 強烈建議設定為 **「固定 IP (Static IP)」**。因為如果它的 IP 每天都在變，其他的伺服器（Agent）就會找不到它要把資料丟去給誰。
   - **實體伺服器 (Agent)** 的 IP 可以是固定的，也可以是 DHCP 自動發派的，**完全不重要！** 就像我們前面提到的，儀表板根本不認識它們的 IP，只要它們能連上中控主機把資料丟進去就好。
