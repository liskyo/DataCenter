# DataCenter 整體前端架構及 UI 畫面與開發環境流程指南

這份文件將作為前端開發者的終極指南，詳細記錄我們為何選擇 **Next.js 16.2.0** 作為框架，以及如何利用 **Tailwind CSS v4** 打造出那套讓人一見鍾情的 **Cyberpunk SCADA (工業監控)** 介面。

---

## 🎨 第一部分：UI 設計哲學 (Design Philosophy)

一般的網頁可能追求留白與簡潔，但身為機房監控系統，我們的設計語言必須是：**「專業、高科技、即時、無死角」**。
因此，我們放棄了常見的明亮主題，全面採用了暗色系的「戰情室美學」。

### 1. 核心配色與識別
- **深空戰艦藍 (`bg-[#010613]`)**：作為全域基底色。相較於純黑，帶有一點點靛藍的深色，能在視覺上降低長時間監看螢幕的疲勞感。
- **神經網路青 (`#06b6d4`, `#4ea8de`)**：這是我們系統的主力裝飾色。所有正常運作的數據、表格框線、圖示，都由這種猶如雷射光束般的青色構成。
- **緊急警報紅 (`#ef4444`, `#881337`)**：只在遇到麻煩（例如高溫、AI 判斷異常）時使用。因為基底是極深的藍色，當這個具有血色霓虹感的紅色一旦亮起並狂暴閃爍 (`animate-pulse`) 時，操作員的眼球能瞬間被吸引。

### 2. 佈景主題的秘密武器 (The Grid Background)
如果你仔細看，會發現所有分頁背後，都有極其細緻的發光網格。
這不是一張圖片，而是用純粹的 CSS 數學刻出來的：
```css
background-image: 
  linear-gradient(rgba(6, 182, 212, 0.05) 1px, transparent 1px),
  linear-gradient(90deg, rgba(6, 182, 212, 0.05) 1px, transparent 1px);
```
這能讓瀏覽器無極限地向左右延伸這個無限畫布，而不損失任何解析度。

## 🛠️ 第二部分：前端技術堆疊 (Frontend Tech Stack)

### 1. Next.js 16.2.0 (App Router 架構)
- **為什麼不用純 React？** 傳統 React 在設定多頁面路由 (`react-router-dom`) 時非常繁瑣。Next.js 的 `App Router` 讓我們只需要建立資料夾 (例：`app/control/page.tsx`)，路由就自動生出來了！這大幅加速了開發速度。
- **"use client" 是什麼？** 這是 Next.js 告訴伺服器「這段程式碼要給瀏覽器自己跑」的指令。因為我們大量依賴 React `useEffect` (例如每秒去後端拉一次資料)，所以所有的監控頁面都在頂端加上了 `"use client"`。

### 2. Tailwind CSS v4 (新世代樣式引擎)
這個專案是在 **Tailwind CSS v4** 環境下建置。請注意與舊版的差異！
在舊版中我們熟悉 `@tailwind base;`，但在本專案的 `globals.css` 裡，我們直接使用了 `@import "tailwindcss";`。
因為我們把預設的 Next.js 自動亮暗切換邏輯拔掉了，所以就算你的 Windows 作業系統是 Light Mode，這個系統也會霸氣地維持暗色科技風，決不跑版。

### 3. Recharts (資料視覺化首選)
要畫出好圖表，我們選了 `recharts`。在狀態總覽 (`/`) 裡，那兩張極其炫酷的 CPU/Temp 面積圖，就是用它畫出來的。
- 繪製原理：後端給我們一個長度為 20 的小陣列，我們把數值倒進 `AreaChart` 中。為了讓畫面美觀，我們給它上了青藍色到透明的漸層遮罩 (`<linearGradient>`)！

### 4. 數位分身與實體編輯模式 (3D Digital Twin & Edit Mode)
前端透過 `@react-three/fiber` 與 `drei` 建構了機房 3D 空間。
- **編輯模式擴充**：支援 `Edit Mode` 切換，操作員可動態新增 RACK、CRAC、PDU、CDU (液冷分配單元)，並且提供 3D 拖曳定位 (`PivotControls`)。
- **持久化儲存 (LocalStorage)**：利用 `zustand/middleware` 裡的 `persist`，所有拖曳好的機房配置會即時寫入瀏覽器快取，F5 重新整理配置也絕不遺失！
- **警報視覺分離**：機櫃骨架顏色僅反映其整體耗電量，而內部伺服器若有高溫，會在機櫃頭頂懸浮 🔥 或是 ⚠️ 符號，達成「環境電力與伺服器散熱」的清晰分離。此外，Dashboard (狀態總覽) 的 2D 矩陣面板也實作了與 3D 完美連動的「青/黃/紅」三段式警戒色與離線實體狀態顯示。

## 🖥️ 第三部分：核心分頁導覽與功用

我們的置頂導覽列 (Navbar) 提供了 7 個獨立的分頁：

1. **📊 狀態總覽 (`/`)**
   - **大腦儀表板**。即時顯示所有伺服器的健康比例 (Doughnut Chart)、CPU 趨勢 (Area Chart)，以及用紅色跟綠色點點標示每台機器的生死狀態。
2. **🎛️ 設備控制 (`/control`)**
   - 模擬重工業開關 (RTU Override)。操作員可以使用 UI 的撥動開關與滑桿，理論上可以送出 `POST` 請求給後端關閉機器電源或調節風扇。
3. **📈 趨勢分析 (`/analysis`)**
   - 介接了 `/history` API，畫出專屬於單台伺服器的長期走勢雙軸曲線圖。
4. **📋 系統日誌 (`/logs`)**
   - 介接了 `/alerts` API，這裡打造成了一個很酷的「終端機命令列面板」，每一秒自動從 MongoDB 抓出最新的資料庫告警日誌。
5. **🔧 維護保養 (`/maintenance`)**
   - 靜態展示待辦的保養工單，利用橘色、紅色等標籤區分輕重緩急。
6. **🌐 網路通訊 (`/network`)**
   - 利用 CSS 與 `lucide-react` 圖示建構出幾何連接線的拓撲圖。
7. **🔒 工程模式 (`/engineering`)**
   - **彩蛋與進階功能區**。點進去會看到恐怖的鮮紅警告畫面，輸入密碼 `admin` 並送出後，React 狀態改變 (`setIsAuth(true)`)，畫面瞬間變成友善的翡翠綠色 Root 控制台。

## 🔄 第四部分：開發環境啟動與除錯 (How to Run & Debug)

### 啟動流程與安裝
1. 打開 PowerShell，進入 `frontend` 資料夾 (`cd frontend`)
2. 安裝裝備：`npm install`
3. 啟動並閃避 Grafana：`npm run dev -p 9001` (指定 9001 Port)
4. 前往 `http://localhost:9001`

### 常見問題與除錯手冊
**Q1：為什麼我的畫布背景變成全白，或者是排版全部擠成一團？**
> **解答**：這是由於 TailwindCSS 的快取常常卡死，或是你在修改 `globals.css` 後沒有讓編譯器意識到。
> 解法：在 PowerShell 視窗按下 `Ctrl+C` 中止，然後刪除隱藏的 `.next` 資料夾，接著再次執行 `npm run dev -p 9001`，保證煥然一新。

**Q2：為什麼一直跳出 `Hydration Error` 的紅字錯誤？**
> **解答**：當你使用 SSR (伺服器渲染) 加載隨機數字或是時區格式不同的時間時，容易發生。不用擔心，只需確保使用 `useEffect` 在客戶端完成掛載後才渲染會跳動的數字即可。

**Q3：我想要擴產，加入第十一台伺服器的面板？**
> **解答**：非常簡單，我們的程式碼是動態渲染的。你只需要到後端 Python 模擬器 `mock_sensor_data.py` 將迴圈改為生成 11 台機器，再到前端重新整理，Next.js 的 `.map()` 陣列渲染魔法就會自動幫你畫出第十一格的圖表！
