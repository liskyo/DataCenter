## 3) 趨勢圖（CPU/溫度）修復

- 檔案：`frontend/src/app/page.tsx`
- 問題：趨勢圖長時間不顯示，追查為 `history` 未穩定累積。
- 修正內容：
  - 將趨勢點累積邏輯改為依 `data` 狀態更新（`useEffect([data])`），避免 callback 時序造成漏寫
  - 2 秒節流新增趨勢點
  - 僅採用可解析的數值（過濾非數字），避免 NaN 汙染圖表
  - `history` 最多保留 30 筆
- 額外處理：
  - 圖表在 `history` 為空時顯示等待文案（`t.awaiting`），避免空白區塊

## 4) SSE/Polling 可靠性強化

- 檔案：`frontend/src/shared/hooks/useSSE.ts`
- 強化內容：
  1. 首次載入先 `pollOnce()` 取得 `/metrics`，避免等不到 SSE 首包時畫面空白
  2. 若 SSE 連線靜默超時，會自動降級至 polling
  3. 進入 polling 後，定時重試 SSE（10 秒一次）
  4. 一旦 SSE 收到訊息，會自動停止 polling 並切回 SSE（`SSE LIVE`）
  5. 以 `useCallback` 固定 `normalizeNodeId / collectKeys`，避免 effect 依賴不穩定導致不必要重連

## 8) 影響範圍與注意事項

- 本次改動集中於前端展示層與資料接收層，未變更後端 API schema。
- `useSSE` 現在是「SSE 優先 + polling 保底 + 自動回切」策略：
  - 若短暫出現 `POLLING` 屬預期容錯行為，不等同系統異常。
- 若未來要調整切換敏感度，可優先調整：
  - `fallbackPollingMs`（目前 5000）
  - SSE 靜默判定時間（目前約 8 秒以上）
  - SSE 重試間隔（目前 10 秒）
