# 3D 動態機房技術差異分析文件 (1150325)

本文件詳細記錄 DataCenter 專案在 3D 動態機房（Digital Twin）功能上，相較於 **2026-03-21** 版本的重大演進與技術革新。

---

## 📊 一、 核心差異對照表

| 功能維度 | 舊版 (2026-03-21) | 新版 (1150325) |
| :--- | :--- | :--- |
| **資料來源** | 前端寫死或簡易 LocalStorage | **佈局驅動 (Layout-Driven)**，讀取 `dcim_layout.json` |

## 🚀 二、 重大技術演進

### 1. 數據驅動的機房空間 (Data-Driven Room)
- **佈局文件 (`dcim_layout.json`)**：現在整個機房的邏輯結構完全由 JSON 定義。包含每個 RACK 的座標、旋轉角度、U 位總量 (uCapacity) 以及內建設備。
- **RoomContext 升級**：環境不再只是黑背景，現在具備 600mm 標準防靜電高架地板 (Antistatic Floor)、壓克力質感牆面、以及帶有刷卡讀卡機的玻璃控制門。

### 2. 精準設備管理 (U-Level Precision)
- **ServerModel**：每一台伺服器現在根據其 `uPosition` 與 `uHeight` 自動計算在機櫃內的垂直位置。
- **EquipmentModel**：引入多樣化的輔助設備（如空調、配電櫃、不斷電系統），讓數位分身更貼近真實機房環境。

### 3. 動態網路佈線 (Cyberpunk Cabling)
- **NetworkLines 組件**：利用 `QuadraticBezierLine` (二次貝茲曲線) 實時計算伺服器櫃與網路櫃 (Network Rack) 之間的連接。
- **視覺效果**：採用虛線 (`dashed`) 與霓虹配色，呈現出高科技、透明且帶有即時感的神經網路拓撲。

## 🛠️ 三、 開發與調整說明

- **如何新增設備？**
  不再需要修改 `App.tsx`，請直接在 `layout/dcim_layout (1).json` 中的 `racks` 或 `equipments` 陣列下新增物件，系統會根據定義自動渲染出對應的 3D 實體。
- **座標與旋轉**：
  座標系統遵循 Three.js 標準 (X, Y, Z)，單位為米 (m)。預設 1U 高度為 0.04445m。
