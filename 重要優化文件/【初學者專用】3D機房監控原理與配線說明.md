# 【初學者專用】3D 機房監控原理與配線說明

這份說明書用簡單方式說明 3D 機房監控資料如何從現場設備進入中控畫面，並已依目前系統現況補充校正。

---

## 1. 它是怎麼運作的？（三步驟）

想象機房是一個人的身體：
1. **資料來源**：設備、Agent、感測器或模擬資料
2. **通訊與處理**：資料送進後端，再交由資料流處理
3. **中控畫面**：前端把收到的結果顯示成 2D 與 3D 畫面

---

## 2. 需要哪些通訊線路？（要插什麼線）

要讓設備「說話」，主要需要以下幾種線：

| 設備名稱 | 需要的線路 | 連接到哪裡？ | 監控什麼？ |
| :--- | :--- | :--- | :--- |
| **Server / Switch** | **網路線 (CAT6 RJ45)** | 機房區交換機 (Switch) | CPU 使用率、溫度 |
| **CDU 液冷主機** | **網路線 (Modbus TCP)** | 管理網路交換機 | 水溫、水壓、水泵轉速 |
| **雙相浸沒機櫃** | **網路線 + 感測器專用線** | 數據採集盒 / PLC | 液面高度、壓力、液體溫度 |
| **環控感測器** | **RS485 或 網路線** | 環控主機 (Gateway) | 漏水警告、環境溫濕度 |

> **簡單結論**：大多數情況下，設備或 Agent 至少需要能透過網路回到中控主機；是否直接抓取或主動上送，取決於設備整合方式。

---

## 3. 各類設備監控重點（白話版）

*   **伺服器 (Server)**：
    *   看它有沒有「發燒」（溫度）？
    *   看它有沒有「很累」（CPU 跑太高）？
*   **CDU 冷卻主機**：
    *   看水夠不夠冷？
    *   看水有沒有在跑（流量）？有沒有漏水？
*   **雙相浸沒式機櫃**：
    *   看裡面的「冷卻液」有沒有乾掉（液位）？
    *   看機櫃裡面的壓力會不會太大？

---

## 4. 為什麼要用 3D 監控？

以前看監控只能看冷冰冰的數字表格。
現在有了 **3D 數位孿生**：
*   **一眼看穿**：哪台機櫃變紅色，就是哪台有問題，不用去現場找。
*   **虛實結合**：畫面上的水流速度跟現場是一樣的，展示起來更有科技感。

---
## 5. 現況補充

- 本專案目前正式主資料流以 `Agent / Device -> HTTP /ingest -> Kafka -> Backend -> SSE -> Frontend` 為主
- 並非所有設備都是由中控主機主動輪詢
- `SNMP`、`Modbus`、`Redfish` 等依設備種類與整合階段決定
