# 3D 機房設備通訊與實時監控技術說明文件

本文件旨在說明 DataCenter 數位孿生系統如何透過後端架構與前端 3D 引擎，達成 Server、Switch、CDU 以及浸沒式冷卻機櫃等設備的實時監控能力。

## 1. 系統架構概覽

監控系統採用 **「端 - 端 - 雲 (Edge-to-Dashboard)」** 的流式數據架構，確保數據從物理設備到 3D 介面的延遲低於 1 秒：

1.  **數據採集層 (Data Acquisition)**：實體設備或模擬代理 (Agent) 透過工業標準協定採集數據。
2.  **核心控制層 (Backend Service)**：以 FastAPI 為核心，處理數據清洗、閥值檢查與警報觸發。
3.  **即時廣播層 (Real-time Broadcast)**：透過 **SSE (Server-Sent Events)** 技術，將數據即時推送至瀏覽器。
4.  **視覺呈現層 (Visual Twin)**：前端 React + Three.js 引擎根據接收到的數據，同步更新 3D 模型狀態與 Dashboard 數值。

---

## 2. 各類設備通訊機制

### A. 伺服器與交換機 (Server & Switch)
-   **通訊協定**：SNMP (Simple Network Management Protocol) 或內建監控代理。
-   **監控指標**：CPU 使用率、核心溫度、網路流量 (Gbps)、進口/出口溫差。
-   **監控策略**：監控主機定時向各節點發送 Get 指令，或由節點在偵測到異常時發送 Trap。在現行系統中，由 `client_agent.py` 模擬這類數據上傳。

### B. CDU 液冷分配單元 (Liquid Cooling Unit)
-   **通訊協定**：**Modbus TCP/IP** 或 **Profinet**。
-   **監控指標**：一次側/二次側供回水溫度、水泵轉速 (RPM)、流量 (LPM)、系統壓力 (bar)。
-   **監控策略**：CDU 通常配備 PLC 控制器，監控主機透過讀取特定暫存器 (Registers) 獲取精密流體數據。

### C. 雙相浸沒式冷卻機櫃 (Immersion Cooling)
-   **通訊協定**：Modbus TCP/IP 或 MQTT (物聯網協定)。
-   **監控指標**：冷卻液高度 (Coolant Level)、儲槽內部壓力、氟化液溫度、熱交換器效率。
-   **監控策略**：由於雙相浸沒涉及相變過度壓力，系統需具備「高頻次採樣」能力（每秒 2-5 次），確保壓力容器安全。

## 3. 即時監控技術核心

### SSE (Server-Sent Events) 廣播機制
不同於傳統的輪詢 (Polling) 會造成網路頻寬浪費，本系統採用 SSE：
-   **單向持續流**：服務器主動向前端「推播」數據。
-   **輕量化**：相比 WebSocket，SSE 在處理大量的設備監控數據時更為穩定且開銷較小。
-   **自動重連**：若網路波動中斷，SSE 會自動嘗試恢復連接，確保監控不間斷。

### 數據映射與 3D 渲染 (Data Mapping)
1.  **ID 匹配**：監控數據帶有 `server_id`，前端接收時會自動匹配 3D 場景中具有相同 ID 的物件。
2.  **動態視覺化**：
    *   **Server**：根據 CPU 負載改變伺服器燈號顏色（藍→黃→紅）。
    *   **CDU**：根據溫度高低改變水流特效的移動速度與顏色深度。
    *   **浸沒槽**：根據液位感測器數值，實時調整 3D 模型中的液面高度 (Liquid Height)。

---

## 4. 異常處理與告警 (Interlocks)

系統在接收數據的當下會進行運算：
-   **閥值攔截**：例如當 CDU 壓力超過 3.5 bar 或浸沒槽液位低於 90% 時，後端 `AppContainer` 會立即觸發 `trigger_alert`。
-   **連動控制**：系統可配置「連動規則」，例如溫度過高時，監控主機自動向 CDU 發送指令調高水泵轉速。
