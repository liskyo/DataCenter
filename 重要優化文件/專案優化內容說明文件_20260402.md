# DataCenter 數位孿生專案 — 2026/04/02 優化內容說明

本次優化重點在於完整整合 **「雙相浸沒式冷卻 (Dual-Phase Immersion Cooling)」** 的監控鏈結，並對儀表板 (Dashboard) 的呈現佈局與系統穩定性進行了深度調整。以下為具體開發內容總結：

## 1. 監控功能整合 (Monitoring Integration)

### 雙相浸沒式監控面版
- **核心指標呈現**：面板整合了進水流量 (Flow Rate)、液氣溫度 (Fluid Temp)、蒸氣壓力 (Vapor Pressure) 與冷卻液位 (Coolant Level) 四大指標。
- **多設備同步映射**：監控列表已由原本的「寫死 (Static)」改為由 「3D 機房 store」 **動態映射**。凡在 Room Editor 內建立的雙相浸沒式機櫃，皆會自動出現在監控面板中，解決了過去僅能顯示單一設備的問題。
- **DEMO 模式支援**：預設在模擬模式下且房內未建立任何浸沒槽時，會自動顯示 `IMM-TAN-001` 的示範數據以供動態展示。

### 特殊視覺狀態
- **精確液位顯示**：在 3D 模型中將冷卻液高度調升至 0.95m，確保實體尺寸 0.8m 的伺服器完全淹沒，符合物理邏輯。
- **一致性佔位符**：切換至「真實數據」或設備離線時，面板數值會顯示 `--` 而非消失或顯示死數據，與系統其他部分（如 CDU）保持視覺一致。

## 2. 數據模擬與代理程式 (Telemetry Simulation)

### 後端模擬器升級
- **全自動預設模擬**：更新了 `backend/services/kafka_service.py`，現在後端模擬器會識別 **`IMM-`** 前綴的設備（如 `IMM-2P-54`），並自動為其產生具備專屬特徵（流量、壓力）的跳動數據。
- **啟動腳本更新**：修改了 `start.bat`，啟動時會自動開啟 `SERVER-015` (標準)、`CDU-001` (液冷) 及 **`IMM-TAN-001` (浸沒)** 三個獨立的模擬 Agent 視窗。

## 3. Dashboard UI/UX 優化

### 佈局調整
- **面版重組**：為了讓左側欄位集中顯示冷卻系統指標，將「設備健康狀態分佈」圓餅圖移至右側欄位，放置於「即時系統警報」下方，補足底部空間。
- **緊湊化處理**：優化了圓餅圖縮放比例，並將該面板的最小高度從 220px 縮減為 **160px**，使整體介面感覺更加緊湊精煉，適合大型螢幕展示。

## 4. 系統穩定性與修復 (Stability Fixes)

### 路由與設定檔修復
- **解決 404 故障**：發現原本專案中並存了 `next.config.mjs` 與 `next.config.ts` 導致 Turbopack 編譯時路由路徑衝突，使其他分頁失效。
- **標準化設定檔**：已統一改用具備最高兼容性的 **ESM 版 `next.config.mjs`**，解決了導航失效與 `exports is not defined` 等環境報錯。
