# 【專家教學】DataCenter CDU 液冷 3D 監控強化與系統優化指南 (2026-04-01)

這份文件將帶領您了解本次系統升級的核心內容。我們不僅強化了視覺效果，更優化了底層數據處理邏輯，讓監控系統更加專業且流暢。

---

## 1. 什麼是 CDU 液冷監控？
**CDU (Cooling Distribution Unit, 冷卻分配單元)** 是數據中心的「心臟」，負責將冷卻液分配到各個伺服器機架中。

### ◈ 3D 監控強化功能：
*   **數位即時儀表板 (Digital HUD)**: 
    - **白話解釋**: 我們在 3D 模型的前方加了一個虛擬螢幕，就像開賽車時看到的抬頭顯示器。
    - **能看到什麼**: 不需要點擊，直接看 3D 畫面就能知道流速 (Flow)、壓力 (Pressure) 與溫度 (Temp)。
*   **X-Ray 透視模式 (透視眼功能)**: 
    - **操作方式**: 當您的滑鼠指到 CDU 設備時，外殼會自動變透明。
    - **內部構造**: 您可以看見內部的「泵浦 (Pumps)」正在旋轉運作，這能讓工程師直觀確認硬體是否正在運轉。
*   **流動動畫同步**: 
    - **視覺效果**: 您會看到從 CDU 連接到機架的管路中有微粒子在移動。
    - **即時連動**: 如果流速變快，粒子跑得就快；流速變慢，粒子也會跟著變慢。

## 2. 全域電源同步 (為什麼設備會「反灰」？)
在實際管理中，確認設備是否「開機」非常重要。

### ◈ 視覺化變動：
*   **什麼是「反灰」(Gray-out)？**: 
    - 當您透過控制面板將伺服器或 CDU **關機 (Power Off)** 時，整個設備在 2D 網頁與 3D 空間中都會褪色變成深灰色。
*   **歸零機制**: 
    - 關機後，系統會自動將 CPU 佔用率與流量數據設為 0。這能防止舊的錯誤數據誤導管理員。
*   **狀態提示**: 
    - 頁面上會明確顯示 **"POWERED OFF"**，讓您一眼看出這是「正常關機」而非「網路斷線 (Offline)」。
*   **設備控制同步**:
    - **新功能**: 當您在「儀表板」關閉一台伺服器時，切換到「設備控制介面」也會看到該伺服器同步變為 "OFFLINE"，不再需要手動更新。
*   **真實模式自動過濾 (Real Mode Logic)**:
    - **原理**: 當您手動切換到「真實模式」時，系統現在能聰明地分出「哪些是模擬的節點」。
    - **效果**: 所有模擬的伺服器會自動被標記為「關機」，3D 機房也會跟著「反黑」。這樣您就能專注於觀察那幾台「真正」有插電且正在運行的硬體（例如您的 Server-15）。

## 3. 系統佈局優化 (看得更清楚)
當機房內有數十台、甚至上百台伺服器同時傳送數據時，原本的圖表會變得很擠。

### ◈ 我們做了什麼優化？
*   **捲軸式 CPU 圖表 (Scrollable Chart)**:
    - **原本狀況**: 節點太多時，側邊的標籤會全部擠在一起。
    - **優化後**: 我們將「各節點 CPU 負載佔比」改為 **動態高度**。現在圖表會根據節點數量自動拉長，並加上捲軸。您可以順暢地上下滑動查看每一台機架的負載，標籤清晰不重疊。
*   **數據更新節流 (Throttling)**:
    - **原本狀況**: 每一秒鐘數據都在跳動，導致瀏覽器拼命重新繪圖。
    - **優化後**: 我們將趨勢更新改為 **2 秒鐘一次**。這在「極速反應」與「電腦負擔」之間取得了完美平衡，讓您能更快看到真實狀態的變化。
*   **真實訊號優先 (Real-Time Sync)**:
    - **原理**: 當多個數據源同時發送同一個 ID 的狀態時，系統現在會自動「優先認可」來自真實 Agent 的訊號，並暫時阻擋模擬器的干擾。這解決了之前 Server-15 status 反應緩慢的問題。
*   **停用多餘動畫**:
    - 我們關閉了圖表切換時的過場動畫。因為即時監控追求的是「快」與「準」，減少動畫能讓頁面切換更加俐落。

## 4. 如何驗證我的系統？
您可以嘗試以下簡單步驟來感受優化成果：
1.  **測試關機**: 點擊任一伺服器的電源按鈕選擇 "Off"。
2.  **觀察 2D**: 看到儀表板中的卡片變灰，曲線歸零。
3.  **觀察 3D**: 切換到 3D 視角，看到對應的機架模型變暗。
4.  **檢查效能**: 嘗試縮放網頁背景，您會發現整體的拖動與響應比以往更加流暢。
