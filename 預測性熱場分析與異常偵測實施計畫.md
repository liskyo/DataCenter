## 🤖 二、 時序預測模型評估與 Big O 效能分析

為保障 Production 環境下的極致性能與推理低延遲（Low Latency），我們對三種主流模型進行了效能與架構利弊評估：

### 📊 模型架構對比與 Big O 分析

| 評估維度 | 解法一：梯度提升樹 (XGBoost / LightGBM) | 解法二：遞歸神經網路 (GRU / LSTM) | 解法三：時序融合變形器 (TFT / Transformer) |
| :--- | :--- | :--- | :--- |
| **訓練時間複雜度** | $O(M \cdot d \cdot N \log N)$<br>*(M: 樹量, d: 深度, N: 樣本數)* | $O(S \cdot H^2 + S \cdot H \cdot I)$<br>*(S: 時間步長, H: 隱藏單元, I: 特徵數)* | $O(N^2 \cdot D)$<br>*(N: 序列長度, D: 投影維度)* |
| **推理時間複雜度** | $O(M \cdot d)$<br>*(極快，與樣本量無關，常數時間)* | $O(S \cdot H^2)$<br>*(隨時間步長線性增長)* | $O(N^2 \cdot D)$<br>*(二次方增長，推理較重)* |
| **優勢 (Pros)** | 1. 結構化表格資料表現極佳<br>2. 推理延遲極低（$< 1\text{ms}$）<br>3. 記憶體開銷小，無需 GPU 加速 | 1. 善於捕捉長期時間依賴性<br>2. 適合非線性動態滯後建模 | 1. 頂級的長期多步預測精度<br>2. 自動捕獲多維特徵動態關聯 |
| **劣勢 (Cons)** | 1. 無法直接處理原始非對齊時序<br>2. 需手工設計滯後特徵 (Lag Features) | 1. 容易梯度消失/過擬合<br>2. 推理開銷較高（無 GPU 時約 $30\text{ms}$） | 1. 訓練需要海量資料與高昂算力<br>2. 邊緣端推理延遲偏高，部署複雜 |
| **Production 推薦**| **首選推薦 (預測 15m - 60m)** | **次選 (適合儲槽液體熱慣性滯後模擬)**| **不推薦 (展場與實時 API 負荷過重)** |

## 💻 三、 3D 動態機房：熱像儀視角 (Thermal Heatmap)

預測數據將直觀地呈現在 DCIM 的 3D 雙生機房中，我們規劃了以下兩種渲染模式：

### 1. 基礎模式：機櫃 Mesh 材質色彩動態映射 (HSL Color Mapping)
系統將每個機櫃（或伺服器節點）的實時與預測溫度，映射至 HSL 色譜中：
*   **藍色 (低溫安全 - 25°C)**：HSL $240^\circ, 80\%, 40\%$
*   **紅色 (高溫警戒 - 85°C)**：HSL $0^\circ, 100\%, 50\%$
*   **轉換公式**（基於 HSL 漸層）：
    $$\text{Hue} = 240.0 - \left( \frac{\text{Temp} - 25.0}{85.0 - 25.0} \right) \times 240.0$$
    用 JavaScript 在 Three.js 逐幀渲染更新：
    ```javascript
    const hue = Math.max(0, Math.min(240, 240 - ((temp - 25) / 60) * 240));
    cabinetMaterial.color.setHSL(hue / 360, 1.0, 0.5);
    ```

### 2. 進階高奢效果：空間立體熱圖 (Volumetric Heatmap Shader)
利用自定義的 **WebGL GLSL Shader**，在 3D 空間中渲染溫度場流動的氣體粒子或漸變力場：
*   **Vertex Shader**：將機櫃熱源點的三維空間坐標傳遞給 Fragment Shader。
*   **Fragment Shader (Ray-marching 或 Distance Fields)**：計算每個空間點到最近熱源點的距離與發熱強度，進行指數衰減渲染：
    ```glsl
    // Fragment Shader 核心片段
    uniform vec3 u_hotspots[10]; // 熱源坐標陣列
    uniform float u_temperatures[10];
    varying vec3 v_worldPosition;
    
    void main() {
        float intensity = 0.0;
        for(int i = 0; i < 10; i++) {
            float dist = distance(v_worldPosition, u_hotspots[i]);
            // 溫度隨距離指數衰減
            intensity += u_temperatures[i] * exp(-dist * 1.5);
        }
        vec3 coldColor = vec3(0.0, 0.2, 0.8);
        vec3 hotColor = vec3(1.0, 0.0, 0.0);
        vec3 finalColor = mix(coldColor, hotColor, clamp(intensity / 85.0, 0.0, 1.0));
        gl_FragColor = vec4(finalColor, 0.35); // 半透明立體熱力雲
    }
    ```

### 3. 前端預測時間滑桿 (Timeline Slider)
*   介面下方配備高質感的玻璃摩砂滑桿，提供選檔：`[ 即時 | +15 分鐘 | +30 分鐘 | +60 分鐘 | +120 分鐘 ]`。
*   拖動滑桿時，前端向 `/api/thermal/predict?horizon=X` 發起輕量級請求，3D 機房材質隨即在毫秒內流暢地轉化為未來預測熱場，供客戶直觀預覽熱點的醞釀過程。

## 🔄 四、 預判型自癒冷卻調度迴路 (Proactive Cooling Loop)

這項預測的核心目的在於**提前干預**，而不是被動等待升溫：

```
                    ┌─────────────────────────┐
                    │ 預測未來 30 分鐘溫度     │
                    └────────────┬────────────┘
                                 │
                                 ▼
                     /───────────────────────\
                    <   預測值 > 55°C (警戒線)  >
                     \───────────────────────/
                                 │ 是
                                 ▼
                    ┌─────────────────────────┐
                    │   智慧主動干預預排      │
                    └────────────┬────────────┘
                                 │
                                 ▼
                    ┌─────────────────────────┐
                    │ 提升 CDU 泵浦轉速至 90%  │
                    │   (提前 20 分鐘冷卻儲槽) │
                    └─────────────────────────┘
```

*   **物理效應**：液冷槽的冷卻液（Coolant）具有很強的「熱慣性（Thermal Inertia）」，從泵浦轉速提高到儲槽實際降溫通常有 15-20 分鐘的物理延遲。
*   **預判控制**：若模型預測 30 分鐘後某 GPU 槽體溫度會突破 55°C，控制中心將立即下發命令，**提早 20 分鐘將對應 CDU 泵浦轉速由 60% 拉升至 90%**，在熱浪到達前先將儲槽降溫，完美將溫度鎖定在安全區內！
